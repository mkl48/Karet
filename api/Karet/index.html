Karet

The Karet entry point. require it on both realms and call Karet.Start once per realm:

local Karet = require(game.ReplicatedStorage.Karet)
Karet.Start({ Toggle = "Semicolon", Theme = "tokyonight", Kaveat = true })

Karet is the product, the terminal surface, and the shell grammar that runs inside it. Register your own commands with Karet.Register, gate them with the permission hooks, and hook the lifecycle with the middleware hooks. The shell, theme engine, editor and rich output toolkit hang off this table too (Karet.Shell, Karet.Theme, ...).

Types​

StartConfig​

</>
interface StartConfig {
Toggle: ToggleSpec?--

key / shortcut to open the terminal, e.g. "Semicolon" or "LeftControl+Semicolon"

Theme: (string | ThemePalette)?--

a built-in theme name or a palette

Placement: ("Window" | "Top" | "Bottom" | "Middle")?--

where the window / bar sits

Compact: boolean?--

single-line command bar instead of the full window

Draggable: boolean?--

drag the Window-placement terminal by its title bar

Parent: Instance?--

where to mount (defaults to PlayerGui)

Size: UDim2?--

override the window size

Position: UDim2?--

override the window position

Kaveat: boolean?--

enable the Kaveat editor

PromptType: ("text" | "window")?--

how Dangerous commands confirm: terminal y/N or an Iris modal

Persist: boolean?--

save / load the profile via Pik DataStores

Admin: boolean?--

enable the moderation suite (kick / ban / mute / audit + dashboard)

BuiltIns: boolean?--

register built-in commands (default true)

DefaultRole: string?--

server: default role for players

RateLimit: RateLimit?--

server: rate limiting (reserved)

KaveatExec: (boolean | (player: Player) → boolean)?--

server: allow kaveat to run code

}

The options table passed to Karet.Start. The same shape is used on both realms; each realm reads the options that apply to it.

Properties​

Version​

This item is read only and cannot be modified. Read Only
</>
Karet.Version: string

The Karet version string, e.g. "0.1.0".

Functions​

RegisterPanel​

Commands
</>
Karet.RegisterPanel(panel: Panel) → Panel

Register a dockable panel, a named view usually built with ctx.Window inside its Open. Panels are surfaced by the panels command and opened with open <name>.

RegisterExtension​

Commands
</>
Karet.RegisterExtension(ext: Extension) → Extension

Register a whole extension: its Commands and Types are wired into Karet, its Panels become openable, and Init(Karet) runs once. This is the unit that makes Karet a platform.

Panels​

Commands
</>
Karet.Panels() → {Panel}

Every registered panel.

SetVar​

Variables
</>
Karet.SetVar(
name: string,
value: any
) → ()

Set a shell variable. The same value is readable in the shell as @name, since this writes straight into the evaluator's scope (Karet.Env). Seed defaults before Karet.Start.

GetVar​

Variables
</>
Karet.GetVar(name: string) → any

Read a shell variable (the @name store).

Run​

Output
</>
Karet.Run(line: string) → any

Run a line through the shell (alias-expanded), exactly as if it had been typed. Returns the chain result { ok, returns?, error? }, or nil before Karet.Start has run.

Toast​

This item only works when running on the client. ClientOutput
</>
Karet.Toast(
text: string,
opts: {
group: string?,
title: string?,
duration: number?
}?
) → ()

Pop a transient toast notification on the active terminal. No-op on the server or before a terminal exists. group keys into the theme like any output group. Handy for extensions and admin events.

UseProfile​

Setup
</>
Karet.UseProfile(name: string) → Profile

Switch the active profile (history, theme, saved kavs, aliases, functions), creating it if it doesn't exist yet. Karet.Profile then points at it.

Register​

Commands
</>
Karet.Register(def: CommandDefinition) → CommandDefinition

Register a command from a table definition. See the commands guide for the full shape (Args, Flags, Modifiers, On, Network, Permission).

Command​

Commands
</>
Karet.Command(name: string) → CommandBuilder

Start the fluent builder: Karet.Command("ban"):Alias("b"):Network("Server"):On(...):Register().

DefineType​

Commands
</>
Karet.DefineType(
name: string,
def: TypeDefinition
) → ()

Register a custom argument type (a la Cmdr). Types parse and validate the raw token into a typed value, and can supply autocomplete.

List​

Commands
</>
Karet.List() → {string}

The names of every registered command.

Get​

Commands
</>
Karet.Get(name: string) → CommandDefinition?

A single command's definition, or nil.

SetRoleHook​

This item only works when running on the server. ServerPermissions
</>
Karet.SetRoleHook(fn: (player: Player) → string) → ()

Decide a player's role. Roles are compared against a command's Permission using the rank ordering from Karet.SetRoles.

SetAuthHook​

This item only works when running on the server. ServerPermissions
</>
Karet.SetAuthHook(fn: (
player: Player,
command: string
) → boolean?) → ()

A per-command gate. Return true to allow, false to deny, or nil to fall through to the normal role check.

SetGroupHook​

This item only works when running on the server. ServerPermissions
</>
Karet.SetGroupHook(fn: (player: Player) → number) → ()

Resolve a player's numeric rank (e.g. a Roblox group rank), for commands that compare against a number.

AllowPlayer​

This item only works when running on the server. ServerPermissions
</>
Karet.AllowPlayer(userId: number) → ()

Allowlist a userId outright.

DenyPlayer​

This item only works when running on the server. ServerPermissions
</>
Karet.DenyPlayer(userId: number) → ()

Denylist a userId outright.

SetDefaultRole​

This item only works when running on the server. ServerPermissions
</>
Karet.SetDefaultRole(role: string) → ()

The role players get when no role hook is set or it returns nothing.

SetRoles​

This item only works when running on the server. ServerPermissions
</>
Karet.SetRoles(ranks: {[string]: number}) → ()

Define the role -> rank ordering used for Permission comparisons, e.g. { User = 0, Mod = 50, Admin = 100 }.

PreExecute​

Middleware
</>
Karet.PreExecute(fn: (ctx: Context) → any?) → ()

Run before every command. Return false to cancel it.

PostExecute​

Middleware
</>
Karet.PostExecute(fn: (
ctx: Context,

) → ()) → ()

Run after every command with its result.

OnErr​

Middleware
</>
Karet.OnErr(fn: (

) → ()) → ()

Run when a command errors.

OnConnect​

This item only works when running on the server. ServerMiddleware
</>
Karet.OnConnect(fn: (player: Player) → ()) → ()

Run when a player joins.

Start​

Setup
</>

Boot Karet. On the client this builds and wires the terminal and the command evaluator and returns the Terminal; on the server it stands up the networking listener and the auth pipeline and returns nil. The same call works in both realms. 

Karet.Start({ Toggle = "Semicolon", Theme = "tokyonight", Kaveat = true })
