Context

The ctx object passed to every command handler. It carries the resolved input (Args, Flags, Modifiers, Input), themed output methods (Print, Success, Err, ...), the return pool (Return), and the rich surfaces (UI, Window, Indicator). 

On = { Server = function(ctx)
    local who = ctx.Args.who
    if ctx.Flags.silent then return end
    ctx.Success(`hi {who.Name}`)
    ctx.Return(who)
end }

Valued long options: --reason=spam -> ctx.Modifiers.reason.

Input​

</>
Context.Input: any

The piped value from the previous command in a pipeline (a | b), or nil.

Player​

</>
Context.Player: Player?

The calling player (nil in Studio / no-player contexts).

UI​

</>
Context.UI: {[string]: any}

The rich-output toolkit: ctx.UI.Table{}, ctx.UI.Toast(), ctx.UI.Link(), ... A no-op off the terminal.

Functions​

Print​

</>
Context:Print(message: any) → ()

Print a line of normal output. Success, Warn, Err, Info, Hint and Reply are the same but route to their own theme groups.

Return​

</>
Context:Return(...: any) → ()

Push return values into the pool, matched positionally to the command's Returns schema. These feed / references, pipelines (a | b), and Karet.Run's result. Nils are preserved.

Abort​

</>
Context:Abort(message: string?) → ()

Cancel the handler mid-run with a message. The command's result becomes { ok = false }, so chains (a && b) see the failure. Unwinds cleanly via a tagged error.
